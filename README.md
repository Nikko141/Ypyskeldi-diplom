# Морфологический анализатор кыргызского языка

Прототип к дипломной работе: математическая модель + Python-реализация + Streamlit-GUI.

## Возможности

- **Анализ словоформ** (`analyze`) — разложение слова на корень и аффиксы с тегами.
- **Генерация словоформ** (`generate`) — построение конкретной формы из леммы и тегов.
- **Парадигмы** — автоматическое склонение и спряжение.
- **Coverage-тестирование** — измерение покрытия на корпусе из ~100 словоформ.
- **Streamlit-интерфейс** с разделами «Анализ», «Генерация», «Парадигма»,
  «База данных», «Тестирование» и «Математическая модель».

## Математическая модель (кратко)

1. **Алфавит и классы фонем.** Гласные делятся на 4 класса гармонии
   `{a, e, o, ö}` по комбинации признаков `palatal × round`.
   Согласные — на `voiceless / voiced / sonorant`.

2. **Сингармонизм.** Каждый аффикс описывается отображением
   `α : C × {a,e,o,ö} → Σ*`, где `C` — контекст последнего согласного основы.
   Класс гармонии слова задаётся последним гласным.

3. **Конечный автомат морфотактики.** Для каждой части речи строится
   DFA `M = (Q, Σ_T, δ, q₀, F)`, где `Σ_T` — алфавит тегов (Pl, Px1Sg, Loc, …).
   Пример для существительного:
   `ROOT → [Pl] → [Poss] → [Case] → END`.

4. **Двухуровневая модель (Koskenniemi 1983, упрощённая):**
   `LEXICAL  бала<Pl><Px1Sg><Loc>  ↔  SURFACE  балаларымда`.

5. **Алгоритм анализа** — рекурсивное снятие суффиксов справа налево
   с проверкой гармонии и продвижением по автомату.

## Результаты тестирования

На тестовом корпусе из 98 словоформ:

| Метрика | Значение |
|---|---|
| Coverage (найден хотя бы один разбор) | **91.8 %** |
| Lemma accuracy | **90.8 %** |
| Roundtrip generate↔analyze (12 примеров) | **92 %** |

Целевой ориентир дипломной работы 60–70 % покрытия превышен.

## Сравнение с существующими системами

- **HFST (Helsinki Finite-State Transducer)** — академический промышленный
  стандарт для тюркских морфологий, основан на полноценной двухуровневой
  модели Koskenniemi. Наш прототип реализует ту же идеологию (FSA + правила
  гармонии), но в учебно-демонстрационном объёме (~300 корней против десятков
  тысяч у HFST) и на чистом Python, без отдельного DSL правил.
- **Apertium-kir** — open-source морфоанализатор на базе lttoolbox/HFST.
  Покрывает гораздо больше лексики и эпигональных моделей; зато наш прототип
  даёт прозрачную пошаговую трассировку и наглядное описание математической
  модели.
